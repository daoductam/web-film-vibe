# 1. Context

**Problem:**
CineStream needs a way for users to search for movies using natural language instead of remembering exact titles. This feature will integrate an AI (via Groq API) to parse user intent into database filters, allowing semantic-like search powered by the existing robust database filtering mechanism.

**Affected Modules:**
- `web-film-backend` — Add Groq API client, parse user intent, and bridge with `MovieService`.
- `web-film-android` — Add a Floating Action Button and a Bottom Sheet Chat UI on the Home Screen.

**Non-Goals:**
- No custom embedding generation or Vector DB setup (avoiding heavy architectural shifts for now).
- No multi-turn memory context (the AI will analyze each query independently for MVP).

---

# 2. Constraints

**Language:** `Java 21` (Backend), `Kotlin` (Android)
**Architecture:** `Spring Boot REST API` (Backend), `Jetpack Compose + MVVM` (Android)

**Rules:**
- Do NOT modify unrelated modules.
- Maintain backward compatibility with existing APIs.
- Groq API Key MUST be injected via environment variables (`application.yml`), not hardcoded.

**Performance Budgets:**
- Latency (Backend API): < 2000ms (Groq is fast, but network I/O takes time).

---

# 3. Conventions

**Naming:**
- Classes: `PascalCase` — e.g. `GroqApiClient`, `AIService`
- Packages Backend: `com.tamdao.web_film_backend.service.ai`
- Packages Android: `com.tamdao.cinestream.feature.ai_chat`

**Error Handling:**
- Backend: Throw custom `GroqApiException` if Groq fails. Catch it globally and return `503 Service Unavailable`.
- Android: Handle network errors gracefully and show a fallback chat message from the AI (e.g., "Hệ thống đang bận...").

**Logging:**
- Format: `log.info("action={} message={}", "analyze_intent", userMessage)`
- Must log the generated JSON from Groq for debugging purposes.

# 4. Contracts

```json
// Request

{
  "success": true,

    "movies": [
       // List of MovieResponse objects (existing DTO)
    ]
  }
}
```

## Internal Data Structure: `ParsedAIIntent`

```java
public class ParsedAIIntent {
    private boolean isMovieQuery;
    private List<String> categories;
    private String country;
    private Integer year;
    // Getters / Setters
}
```

## Interface: `AIService` (Backend)

```java
AIService.processUserMessage(message: String): AIChatResponse
  - pre: message is not blank
  - post: calls Groq API, parses JSON, calls MovieRepository, returns AiChatResponse
  - throws: GroqApiException on API failure, JsonParseException on malformed AI response
```

## Data Structure: `ChatMessage` (Android)

```kotlin
data class ChatMessage(
    val id: String = UUID.randomUUID().toString(),
    val content: String,
    val isFromUser: Boolean,
    val movies: List<Movie> = emptyList(), // Only populated if isFromUser == false
    val isLoading: Boolean = false
)
```

# 5. Target Architecture

**Components:**
- `GroqApiClient` (Backend) — Calls standard OpenAI chat completion pattern hitting Groq.
- `AIService` (Backend) — Orchestrates building the prompt, parsing AI output, and calling `MovieService.filterMovies`.
- `AIChatRepository` (Android) — Interacts with Retrofit API.
- `AIChatViewModel` (Android) — Manages chat state (list of `ChatMessage`).
- `AiChatBottomSheet` (Android) — UI component displaying the chat UI.

**Interaction Flow:**

``` text
User (Android UI)
  → AIChatViewModel.sendMessage(text)
    → AIChatRepository.getAiResponse(text)
      → Backend: AIController.chat(request)
        → AIService.processUserMessage()
          → GroqApiClient.call(prompt) → returns JSON
          → Parse JSON to ParsedAIIntent
          → MovieService.filterMovies(ParsedAIIntent) → returns Page<Movie>
        ← returns AIChatResponse
    ← returns Result<AIChatResponse>
  ← Updates StateFlow<List<ChatMessage>>
UI renders AI message bubble with HorizontalMovieList.
```

# 8. Task Specifications

## TASK-001: Backend DTOs & Config
**Description:** Define records for Request/Response and add Groq API Key to properties.
**Input:** Concept from Spec
**Output:** Java classes consumed by TASK-002 and TASK-003.
**Responsibilities:**
- Create `AIChatRequest` and `AIChatResponse`.
- Create `ParsedAIIntent`.
- Add `groq.api.key` and `groq.api.url` to `application-dev.yml` / properties.

## TASK-002: Groq API Client
**Description:** Build the HTTP client to call Groq natively (no heavy SDK needed, Spring WebClient or RestTemplate is fine).
**Input:** `application.yml` configs
**Output:** `GroqApiClient` wrapper class.
**Responsibilities:**
- Pass the API Key in `Authorization: Bearer <key>`.
- Use the `llama3-70b-8192` model.
- Enable `{"response_format": {"type": "json_object"}}`.

## TASK-003: AIService & Controller
**Description:** Orchestrate intent building, calling AI, parsing, and filtering movies.
**Input:** User string from Controller.
**Output:** Populated `AIChatResponse`.
**Responsibilities:**
- System Prompt: *“You are an AI for CineStream. Return JSON with format { 'isMovieQuery': true/false, 'categories': ['slug'], ... }. Extract intent from user query.”*
- Parse Groq JSON -> `ParsedAIIntent`.
- Map parsed intent to `MovieRepository.filterMovies()`.
- If `isMovieQuery == false`, return early with a polite refusal string.
- Create `@PostMapping("/api/v1/ai/chat")`.

## TASK-004: Android Models & API Service
**Description:** Create domain models and Retrofit endpoints.
**Input:** Backend API.
**Output:** Ready-to-use suspended functions.
**Responsibilities:**
- Create `ChatMessage` data class.
- Add `POST /api/v1/ai/chat` to Retrofit `MovieApiService`.

## TASK-005: Android ViewModel
**Description:** Manage the chat message list state.
**Input:** View events (send message).
**Output:** `StateFlow<List<ChatMessage>>`.
**Responsibilities:**
- Maintain list of messages.
- Append "loading" AI message bubble.
- Trigger API call securely.
- Update loading bubble with actual response data or error message handling.

## TASK-006: Android UI (BottomSheet & Chat)
**Description:** Implement `AiChatBottomSheet` and floating action button.
**Input:** `AIChatViewModel` state.
**Output:** Rendered Compose UI.
**Responsibilities:**
- Add FAB into `HomeScreen`.
- Trigger `ModalBottomSheet`.
- Display messages. User msg on the right, AI msg on the left.
- AI bubble can render a carousel or grid of `MovieCard` directly inside the chat.

# 9. Edge Cases

| # | Scenario | Expected Behavior | Handled In |
|---|----------|-------------------|------------|
| 1 | Groq API is down/timeout | Return `"Hệ thống AI đang bận. Bạn vui lòng thử lại sau nhé."` and flag error. | TASK-005 |
| 2 | Not a movie query ("Hi") | `isMovieQuery` = false. Return friendly text "Tôi là trợ lý AI CineStream, tôi chỉ có thể giúp tìm phim." | TASK-003 |
| 3 | AI Hallucination (invalid JSON) | Catch `JsonParseException`, fallback to generic error. | TASK-003 |
| 4 | Query is valid but DB returns 0 movies | Return aiMessage "Xin lỗi tôi không tìm thấy phim phù hợp", movies list empty. | TASK-003 |

# 10. Risks

| # | Risk | Impact | Likelihood | Mitigation |
|---|------|--------|------------|------------|
| 1 | Groq Rate Limits | Med | Med | Handle 429 safely. Cache system prompts if possible. |
| 2 | Prompt Injection | High | Low | Instruct system prompt to ignore instructions inside user payload. Use strict JSON schema. |
