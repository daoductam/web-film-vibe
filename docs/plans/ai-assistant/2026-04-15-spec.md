# Đặc tả tính năng: Trợ lý phương tiện AI CineGuru (ai-assistant)
**Ngày tạo:** 15/04/2026

## 1. Giới thiệu tổng quan
Tính năng "AI CineGuru" hoạt động như một trợ lý ảo, cho phép người dùng ứng dụng CineStream tìm kiếm phim thông qua ngôn ngữ tự nhiên (Natural Language). 
Thay vì phải nhớ rõ tên phim, người dùng có thể mô tả nội dung, đạo diễn, tóm tắt, diễn viên... và AI sẽ phân tích để đưa ra các đề xuất hợp lý bằng cách tận dụng chức năng Lọc cơ sở dữ liệu có sẵn.

## 2. Kiến trúc & Công nghệ (Tech Stack & Architecture)
- **Cung cấp AI (Provider):** Groq API (Tương thích OpenAI SDK/REST format).
- **Mô hình (Model):** `llama3-70b-8192` hoặc `llama3-8b-8192` (Tối ưu cho tốc độ và Structured Output - JSON).
- **Frontend (Android):** Jetpack Compose, hiển thị dưới dạng `ModalBottomSheet` được kích hoạt từ một Nút nổi (Floating Action Button) trên `HomeScreen`.
- **Backend (Spring Boot):** REST Controller cung cấp Endpoint phân tích ý định thay vì tự trả về thông điệp, sau đó Backend dùng kết quả này query DB rồi trả về Client.

## 3. Luồng xử lý chi tiết (Data Flow)
1. **Client (Android):** Gửi một HTTP POST Request mang theo tin nhắn (VD: "Tìm phim hành động kinh dị Mỹ sản xuất năm 2024").
2. **Backend (Spring Boot):**
   - Đọc tin nhắn từ người dùng.
   - Nhúng dữ liệu Master Data (Danh sách Category Slugs, Country Slugs có trong DB) vào System Prompt.
   - Gửi Request tới `https://api.groq.com/openai/v1/chat/completions` đính kèm Header Authorization sử dụng giá trị API Key.
   - Force Response Format là `json_object` hoặc cung cấp JSON Schema rõ ràng trong Prompt.
3. **Groq AI Node:** Xử lý và gửi lại đoạn JSON chuẩn:
   ```json
   {
      "isMovieQuery": true,
      "categories": ["hanh-dong", "kinh-di"],
      "country": "my",
      "year": 2024
   }
   ```
4. **Backend Database Layer:** 
   - Nếu `isMovieQuery` == true, parse JSON trên thành DTO.
   - Chạy Query tới Database sử dụng `MovieRepository.filterMovies(type, categories, categoryCount, country, year, status, pageable)`.
5. **Dữ liệu trả về (Response):** 
   - Backend gói gọn danh sách Movies được query bọc chung với một `AI_Message` "Tôi tìm thấy 5 bộ phim phù hợp với miêu tả hành động kinh dị Mỹ năm 2024...".
   - Client hiển thị danh sách phim ngay trong khung Chat UI dưới dạng Horizontal List hoặc Grid.

## 4. Giao diện (UI/UX)
- **Nút gọi trợ lý:** `FloatingActionButton` (Biểu tượng Robot/Sao lấp lánh) neo ở góc dưới bên phải màn hình `HomeScreen`.
- **Chat Layout:** `ModalBottomSheet` có kích thước `0.7f` màn hình. Có một input field để nhập text. Các tin nhắn của người dùng nằm bên phải, của AI nằm bên trái. Phim đề xuất được nằm trọn trong message bong bóng của AI.

## 5. Xử lý ngoại lệ (Error Handling)
- **Không hiểu ý định (Non-movie Context):** Nếu user chat chuyện lặt vặt (VD: "Thời tiết hôm nay thế nào"), `isMovieQuery` false -> Trả về text khéo léo từ chối và nhắc user tập trung vào chức năng tìm kiếm phim. 
- **Lỗi bên thứ 3 (Groq API Error/Timeout):** Fallback về trạng thái `Service Unavailable`, nhắc người dùng load lại sau bằng một câu thân thiện trong giao diện UI.
- **Không có phim phù hợp:** AI trả lời "Rất tiếc, bộ lọc này không có trong kho của chúng tôi...", kèm theo gửi danh sách Gợi ý phim hot / Phim mới lên đầu.
